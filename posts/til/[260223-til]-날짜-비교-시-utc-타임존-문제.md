<h2 data-ke-size="size26">의 함정과 올바른 날짜 비교</h2><h2 data-ke-size="size26">1. 문제의 핵심: 과 로컬 타임존의 괴리</h2><p data-ke-size="size16">자바스크립트의 <code>Date</code> 객체는 내부적으로 유닉스 타임스탬프(UTC 기준 밀리초)를 저장합니다.<br>문제는 이를 문자열로 변환할 때 발생합니다.</p><h3 data-ke-size="size23">❌ 흔한 실수:  절삭</h3><p data-ke-size="size16">많은 경우 <code>YYYY-MM-DD</code> 형식의 날짜가 필요할 때 다음과 같은 코드를 사용합니다.</p><pre class="reasonml"><code>new Date().toISOString().slice(0, 10);
</code></pre><p data-ke-size="size16">&nbsp;</p><p data-ke-size="size16"><b>이 방식은 한국(UTC+9) 기준 오전 00:00 ~ 08:59 사이에 "어제" 날짜를 반환합니다.</b><br><code>toISOString()</code>은 이름 그대로 <b>UTC(0시)</b> 기준의 표준시를 출력하기 때문입니다.</p><table style="border-collapse: collapse;" border="1" data-ke-align="alignLeft"><tbody><tr><td><b>출근 직후</b></td><td>2026-02-23 <b>08:30</b></td><td>2026-02-22 <b>23:30</b></td><td><b>"2026-02-22" (오류)</b></td></tr><tr><td><b>점심 시간</b></td><td>2026-02-23 <b>12:30</b></td><td>2026-02-23 <b>03:30</b></td><td>"2026-02-23" (정상)</td></tr></tbody></table><hr data-ke-type="horizontalRule" data-ke-style="style1"><h2 data-ke-size="size26">2. 날짜 처리의 3가지 원칙</h2><p data-ke-size="size16">타임존 이슈를 근본적으로 방지하기 위해 다음 원칙을 준수해야 합니다.</p><h3 data-ke-size="size23">원칙 1: 날짜 비교는 '문자열' 혹은 '순수 날짜 객체'로 통일</h3><p data-ke-size="size16">시간 정보(시, 분, 초)가 포함된 상태로 비교하면 타임존 오프셋에 따라 결과가 뒤틀립니다. 날짜만 비교해야 한다면 반드시 <code>YYYY-MM-DD</code> 포맷의 <b>로컬 문자열</b>로 변환 후 비교하세요.</p><h3 data-ke-size="size23">원칙 2:  메서드 사용 지양</h3><p data-ke-size="size16">"오늘"의 연, 월, 일을 구할 때는 사용자의 환경(브라우저) 또는 서버 설정에 맞춘 로컬 메서드를 사용해야 합니다.</p><ul style="list-style-type: disc;" data-ke-list-type="disc"> 
 <li><b>사용 금지:</b> <code>getUTCFullYear()</code>, <code>getUTCMonth()</code>, <code>getUTCDate()</code></li> 
 <li><b>사용 권장:</b> <code>getFullYear()</code>, <code>getMonth()</code>, <code>getDate()</code></li> 
</ul><h3 data-ke-size="size23">원칙 3: 서버와 클라이언트의 '오늘' 정의 동기화</h3><p data-ke-size="size16">서버가 UTC 기준인 DB 환경에서 그대로 "오늘"을 계산하면, 한국 클라이언트가 보는 "오늘"과 데이터가 불일치하게 됩니다. 기준이 되는 타임존을 명시하거나, 날짜 문자열(<code>YYYY-MM-DD</code>) 자체를 주고받는 것이 안전합니다.</p><hr data-ke-type="horizontalRule" data-ke-style="style1"><h2 data-ke-size="size26">3. 추천 해결 방법 </h2><h3 data-ke-size="size23">A. 유틸리티 함수 활용</h3><p data-ke-size="size16">외부 라이브러리 없이 로컬 날짜를 안전하게 추출하는 방법입니다.</p><pre class="qml"><code>/**
 * 전달받은 Date 객체를 해당 환경의 로컬 날짜(YYYY-MM-DD)로 변환
 */
export function toLocalDateStr(date = new Date()) {
  const y = date.getFullYear();
  const m = String(date.getMonth() + 1).padStart(2, "0");
  const d = String(date.getDate()).padStart(2, "0");
  return `${y}-${m}-${d}`;
}
